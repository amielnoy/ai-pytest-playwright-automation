# Session 26 - Exercises

## Exercise 1: Write A Codex Prompt

Write a Codex CLI prompt for adding one new web UI test. Include goal, scope, constraints, verification, and expected output.

## Exercise 2: Inspect Before Editing

List the exact commands Codex should run before editing a test file. Explain what each command proves.

## Exercise 3: Verification Plan

Create a verification plan for a page-object change. Include one focused unit test command and one browser test command.

## Exercise 4: Approval Boundary

Identify three commands that should require approval in a local QA automation workflow. Explain why.

## Exercise 5: Review The Diff

After Codex changes a file, write a short review checklist covering scope, duplication, assertions, fixtures, and reporting.
