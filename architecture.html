הפרויקט הוא מסגרת בדיקות אוטומטיות רב-שכבתית שנבנתה עבור אתר ה-demo של TutorialsNinja — אתר e-commerce לדוגמה.

הארכיטקטורה מיישמת את פירמידת הטסטים (testing pyramid) עם ארבע שכבות ברורות: Unit → Contract → API → UI, כאשר כל שכבה בנויה על הקודמת ומספקת כיסוי שונה ומשלים.

טכנולוגיות ליבה

🎭
Playwright (Python)
אוטומציה של דפדפן Chromium לטסטי ה-UI. כל בדיקה פועלת בהקשר (context) נפרד.
🧪
pytest + pytest-xdist
מנוע הרצה ראשי. תמיכה ב-parallel execution עם -n auto --dist loadscope.
📊
Allure 3
דוחות עשירים עם היסטוריה, screenshots בכישלון, ו-GitHub Pages deployment.
⚡
FastAPI Server
שרת אוטומציה מקומי: test-data API, הרצת pytest, דוחות, ו-mock store.
🐳
Docker
סביבת הרצה מבודדת וניתנת לשחזור עבור CI וסביבות ייצור.
🔄
GitHub Actions
CI/CD: הרצת טסטים → יצירת דוח Allure → פרסום ב-GitHub Pages.

📄 Page Object Model

כל האינטראקציה עם הדפדפן כתובה בקלאסי POM תחת pages/. הטסטים עצמם לא מכילים שום selector ישיר — רק קריאות לפונקציות ה-page objects.

BasePage ← pages/base_page.py
├── self.alert — AlertComponent זמין לכל הדפים (ירושה)
├── navigate(path, wait_until) — ניווט בסיסי
├── title @property — כותרת הדף הנוכחי
├── url @property — URL הדף הנוכחי
├── wait_for_visible(selector, timeout?) — ציפייה לאלמנט גלוי
├── take_screenshot() → bytes — צילום מסך
└── get_error_message() — שגיאת banner/field (דרך self.alert)
HomePage ← pages/home_page.py
├── open()
├── search(query)
├── go_to_login() / go_to_register()
└── is_logged_in() / logout()
SearchResultsPage ← pages/search_results_page.py
├── product_names()
├── get_products_under_price(query, max_price, limit)
├── add_items_to_cart(products) — משתמש ב-self.alert (בסיס)
├── sort_by_name_ascending()
├── stored_product_information()
└── wait_for_product_results() — קורא ל-wait_for_visible (בסיס)
CartPage ← pages/cart_page.py
├── open()
├── get_cart_total()
└── is_empty()
RegisterPage ← pages/register_page.py
├── register(first_name, last_name, email, …)
├── is_registration_successful()
└── get_error_message() ← ירושה מ-BasePage
LoginPage ← pages/login_page.py
├── login(email, password)
├── is_login_successful()
└── get_error_message() ← ירושה מ-BasePage
Components (pages/components/) — ירושה מ-BaseComponent, ללא __init__ מיותר
├── NavBarComponent — חיפוש, ניווט, login status
├── AlertComponent — banner ושגיאות inline (חי ב-BasePage)
├── CartSummaryComponent — תקציר עגלה

ProductInfo / StoredProductInfo (dataclasses) ← pages/models.py
└── name, price, index / name, picture_url, description, price
💡
כלל ברזל: selectors (CSS, XPath, text) נכתבים אך ורק בתוך ה-page objects. טסט שמכיל page.locator() ישירות — לא עבר review.

כל הטסטים כתובים עם Allure decorators לדוח עשיר ואינפורמטיבי.

Decorators שבשימוש

Decoratorתפקידערכים לדוגמה
@allure.featureקיבוץ טסטים לפי featureSearch, Cart, Registration, API Tests, Contract Tests
@allure.storystory בתוך featureSearch with price filter, addItemsToCart
@allure.titleשם קריא לאדם"Search returns 200 and non-empty HTML"
@allure.severityרמת חשיבותCRITICAL, NORMAL
with allure.step(...)שלבים פנימיים בטסטמה שרואים בדוח כ-nested steps
allure.attach(...)צירוף artifactsscreenshots, JSON, TEXT

הרצה מקומית

# הרצה עם שמירת תוצאות
pytest

# יצירת דוח
npm run allure:generate

# פתיחת דוח בדפדפן
npm run allure:open
