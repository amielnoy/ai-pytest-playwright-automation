הפרויקט הוא מסגרת בדיקות אוטומטיות רב-שכבתית שנבנתה עבור אתר ה-demo של TutorialsNinja — אתר e-commerce לדוגמה.

הארכיטקטורה מיישמת את פירמידת הטסטים (testing pyramid) עם ארבע שכבות ברורות: Unit ← Contract ← API ← UI, כאשר כל שכבה בנויה על הקודמת ומספקת כיסוי שונה ומשלים.

Class data members for locators

כל locator או ערך שמזהה אלמנט נשמר כ-member קבוע במחלקה: role, name, label, placeholder, selector, route, text קבוע, attribute name וכדומה. המתודות קוראות ל-members האלה, ולא מגדירות strings חדשים inline. BasePage עצמה עוקבת אחרי הכלל — הסלקטורים "html" ו-"[id]" הוגדרו כ-_HTML_SELECTOR ו-_ANY_ID_SELECTOR.

class BasePage:
    _HTML_SELECTOR = "html"
    _ANY_ID_SELECTOR = "[id]"

    def __init__(self, page, base_url):
        self._html = page.locator(self._HTML_SELECTOR)
        self._id_elements = page.locator(self._ANY_ID_SELECTOR)

class LoginPage(BasePage):
    _EMAIL_ROLE = "textbox"
    _EMAIL_NAME = "E-Mail Address"
    _PASSWORD_LABEL = "Password"

    def login(self, email, password):
        self.page.get_by_role(self._EMAIL_ROLE, name=self._EMAIL_NAME).fill(email)
        self.page.get_by_label(self._PASSWORD_LABEL).fill(password)

test_accessibility_sanity.py — בדיקות sanity לנגישות (10 tests)

♿
בדיקות sanity נגישות חיוביות ללא axe חיצוני. הן בודקות יכולת בסיסית של הדפים הקריטיים להיחשף נכון לקורא מסך ולניווט סמנטי, בלי להפוך את suite ה-smoke ל-audit נגישות מלא.

מטרת בדיקות הנגישות היא לזהות במהירות שבירות בסיסיות שמפריעות לשימוש באתר דרך טכנולוגיות מסייעות: שדות בלי label, כפתורים או קישורים בלי שם נגיש, תמונות מוצר בלי alt, duplicate ids, או טקסטי שגיאה שלא חשופים כטקסט נראה. הבדיקות רצות כחלק מ-sanity ולכן הן קצרות, ממוקדות בדפים הקריטיים, ולא תלויות בכלי חיצוני.

הגישה המועדפת היא להשתמש ב-locators הסמנטיים של Playwright כמו get_by_role, get_by_label, get_by_placeholder ו-get_by_text. זה נותן שתי שכבות ערך: הטסט קריא יותר, ובאותו זמן הוא מאמת שהאלמנט באמת נגיש לפי שם, label או role שהמשתמש אמור לקבל גם דרך reader או ניווט מקלדת.
