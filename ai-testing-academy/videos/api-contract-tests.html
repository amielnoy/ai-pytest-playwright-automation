The Test Pyramid

📜 Contract
🔌 API
⚡ Unit — many & fast

Fewer expensive UI tests on top. More coverage in the fast layers below.

The Rule

No raw requests in tests

# services/rest_client.py handles retries + timeout
# one service class per endpoint
results = search_service.search("MacBook")
assert results.status_code == HTTP_OK  # from constants

Change the API in one place — not in fifty tests.

Data-Driven 
@pytest.mark.parametrize("query", ["MacBook", "iPhone", "Samsung"])
def test_search(search_service, query: str):
    hits = search_service.search(query)
    assert hits.count() > 0

One function → a dozen cases. Status codes live in one constants file.
