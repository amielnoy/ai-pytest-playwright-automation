Automation QA Course

Manual testing to production-grade automation, AI workflows, CI, and observability.

A Git-ready 29-session learning path that starts with test design fundamentals and ends with a Docker Compose, Grafana, Allure, Playwright, pytest, API, and AI-assisted QA automation stack.

Learning Phases

The course is designed to be followed in order. Each phase adds a capability that the next phase depends on.

Sessions 1-3

Foundations

Manual QA, AI-assisted test thinking, Python basics, and the habits needed before browser automation.

After Session 3

Git Basics

Repository workflow, branches, commits, diffs, pull requests, merge conflicts, and safe collaboration habits.

Sessions 4-13

Automation Core

Playwright, locators, exceptions, pytest fixtures, API testing, database testing, POM, and debugging.

Sessions 14-22

AI, CI, and Review

AI tooling, agents, failure analysis, reporting, Playwright CLI workflows, and automation code review.

Session 23

Capstone

Production-level framework delivery with clear architecture layers, ADRs, pre-commit checks, and CI quality gates.

Sessions 24-29

Advanced Tooling

Claude Code, skills, plugins, connectors, Codex CLI, Ollama local AI, Docker Compose, Prometheus, and Grafana.

Git Basics Teaching Block

Use this short module after Session 3, once students have written basic Python and before they start editing browser automation files. The goal is practical confidence, not full Git theory.

1. OrientExplain repository, working tree, staging area, commit history, remote, and branch in plain language.
2. InspectPractice `git status`, `git diff`, and reading changed files before editing or committing.
3. IsolateCreate a branch per task, keep edits focused, and avoid mixing course notes with framework changes.
4. CommitStage intentional files, write a clear commit message, and verify the diff before pushing.
5. RecoverHandle common mistakes: unstaged changes, merge conflicts, wrong branch, and pull before push.

How to Use Each Session

Use the same loop for every topic so learning, practice, and verification stay connected.

1. ReadStart with `lecture.md` to understand the concept and quality bar.
2. RunExecute `lecture.py` or the narrow session tests when available.
3. InspectRead the supporting files and matching `course/framework/` layer.
4. PracticeComplete `EXERCISES.md` with concrete evidence and review notes.
5. VerifyRun the narrowest relevant tests before moving to the next session.

This overview links to teaching material under `course/`. Capstone deliverables belong in the production directories: `pages/`, `services/`, `flows/`, `tests/`, `data/`, and configuration files.
